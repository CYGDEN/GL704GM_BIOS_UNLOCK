## Почему этот метод необходим? (Проблема VBS и FACEIT)

В современных сборках Windows включена функция **VBS (Virtualization-Based Security)**, которая обязательна для работы античитов (например, **FACEIT AC** или **Vanguard**). 

* **Проблема:** При активном VBS операционная система полностью блокирует утилитам вроде `ThrottleStop` или `Intel XTU` доступ к регистрам управления напряжением процессора. Программный андервольтинг из-под Windows становится невозможен.
* **Решение:** Данный репозиторий позволяет обойти это ограничение. Мы меняем переменные вольтажа напрямую в NVRAM через UEFI GRUB Shell до загрузки ОС. Windows с активным VBS принимает эти настройки как «родные» параметры железа, а вы получаете сниженные температуры в играх без конфликтов с античитом.
